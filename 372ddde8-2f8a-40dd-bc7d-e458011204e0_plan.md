# Project Plan: Inventory, Procurement (P2P), Order-to-Cash (O2C), Task, Project, Asset, Visitor

**Scope Summary**
- Deliver ERP-style modules integrated into Fawnix-Verse for inventory, procurement-to-purchase, order-to-cash, task management, project management, asset management, and visitor management.
- Use existing service boundaries where possible and add new services only when needed for clear ownership.

**Assumptions**
- 2-week sprints, team of 6-8 engineers, 1 PM/BA, 1 UX, 2 QA.
- Existing auth and user management in `backend/services/identity-service/`.
- Service-per-domain architecture with REST APIs and per-service database migrations.
- Web client lives in `src/` (Vite/React) and will expose new modules via the main navigation.

**Functional Requirements by Module**
**Inventory Management**
- Item master with categories, UoM, variants, barcodes, lot/serial, safety stock.
- Warehouse and location hierarchy, bins, multi-warehouse transfers.
- Stock transactions: receipt, issue, transfer, adjustment, and reservations.
- Cycle counts and reconciliation with audit trail.
- Reorder rules, min-max levels, and alerts.
- Inventory valuation (FIFO, weighted average) with cost history.
- Batch expiry, quality hold, and quarantine.
- Inventory KPIs and dashboards.

**Procurement to Purchase (P2P)**
- Vendor onboarding, verification, and compliance documents.
- Price lists, contracts, and preferred vendor rules.
- Purchase requisitions with approval workflows.
- RFQ/RFP creation, vendor comparison, and award.
- Purchase orders, change orders, and releases.
- Goods receipt, inspection, and put-away.
- 2-way and 3-way invoice matching.
- Returns to vendor and debit notes.
- Spend analytics, vendor scorecards, and lead time tracking.

**Order to Cash (O2C)**
- Customer master, price books, discounts, and tax profiles.
- Quote to sales order conversion with approvals.
- Inventory allocation, pick-pack-ship, and shipment tracking.
- Delivery notes and proof of delivery.
- Invoicing with tax rules and credit notes.
- Payment capture, refunds, and reconciliation.
- AR aging, dunning, and collections workflows.
- Returns (RMA), replacements, and warranty claims.
- Sales analytics, margin tracking, and pipeline reporting.

**Task Management**
- Task creation, assignment, priority, due dates, and statuses.
- Kanban and list views with filters and saved views.
- Subtasks, checklists, and dependencies.
- Comments, mentions, and attachments.
- Reminders, SLA rules, and escalation.
- Time tracking and workload reporting.
- Recurring tasks and templates.

**Project Management**
- Project setup with phases, WBS, and milestones.
- Resource allocation and capacity planning.
- Budgeting, cost tracking, and forecasts.
- Timesheets and approvals.
- Risk, issue, and change management.
- Progress tracking with baselines and variance.
- Client billing for time and fixed-price projects.
- Project reporting and health dashboards.

**Asset Management**
- Asset register with tags, categories, and locations.
- Lifecycle tracking from acquisition to disposal.
- Assignment, transfer, and custody history.
- Preventive and corrective maintenance schedules.
- Work orders, spare parts usage, and service history.
- Depreciation schedules and valuation reports.
- Physical verification and audit trails.
- Warranty, insurance, and compliance tracking.

**Visitor Management**
- Visitor pre-registration and approval workflows.
- ID verification and document capture.
- Check-in/out, badge printing, and host notifications.
- Visitor logs with search and export.
- Blacklist/whitelist and security alerts.
- Multi-site support and zone-based access.
- Emergency evacuation list and compliance reports.
- Data retention and privacy controls.

**Cross-Cutting Requirements**
- Role-based access control, org hierarchy, and multi-site permissions.
- Configurable workflow/approval engine.
- Audit trail for all critical transactions.
- Document management and attachments across modules.
- Notification service (email/SMS/WhatsApp) and templating.
- Reporting layer with dashboards and scheduled exports.
- Import/export utilities and API-first design.
- Localization (currency, tax, language) where required.

**High-Level Data Model**
- Parties: vendors, customers, employees, visitors.
- Core masters: items, warehouses, assets, projects.
- Transactions: PR, RFQ, PO, GRN, SO, shipment, invoice, payment.
- Task and project artifacts: tasks, milestones, timesheets, risks.

**Timeline (Indicative, 2-Week Sprints)**
| Phase | Weeks | Deliverables |
| --- | --- | --- |
| 0 | 1-2 | Discovery, process maps, detailed PRD, backlog, KPIs |
| 1 | 3-5 | Platform foundation: workflow engine, audit, docs, notifications |
| 2 | 6-11 | Inventory + P2P MVP (core flows end-to-end) |
| 3 | 12-17 | O2C MVP (quote to cash with AR) |
| 4 | 18-22 | Task + Project MVP |
| 5 | 23-27 | Asset + Visitor MVP |
| 6 | 28-30 | Hardening, UAT, training, go-live |

**Milestones**
- M1: Approved PRD and solution architecture.
- M2: Shared services live (workflow, notifications, audit).
- M3: Inventory + P2P live with core reports.
- M4: O2C live with invoicing and payments.
- M5: Task + Project live with timesheets.
- M6: Asset + Visitor live with compliance reports.
- M7: Production readiness and rollout.

**Critical Paths / Areas to Modify**
- `backend/services/inventory-service/` for inventory and procurement APIs, domain models, and workflows.
- `backend/services/sales-service/` for O2C APIs, pricing, invoicing, and AR.
- `backend/services/hrms-service/` for task and project management, or a new service if separation is required.
- `backend/services/platform/` for shared workflow, notification, and audit components.
- `backend/services/identity-service/` for RBAC changes and org hierarchy.
- `src/` for UI modules, routing, shared components, and dashboards.
- `docker/` and `compose.yml` for local dev and service wiring.
- `backend/services/*/src/main/resources/` for DB migrations and seed data.

**Verification Plan**
- Unit tests for domain logic and workflow rules in each service.
- Integration tests for API contracts and cross-service flows.
- End-to-end flows:
- Inventory receipt -> stock update -> reservation -> sales order allocation.
- PR -> PO -> GRN -> invoice match -> payment.
- Quote -> SO -> shipment -> invoice -> payment -> AR aging.
- Task -> project milestone -> timesheet approval.
- Asset maintenance -> work order -> spare consumption.
- Visitor pre-approval -> check-in/out -> audit report.
- Performance tests for peak order processing and inventory adjustments.
- UAT with real business scenarios and sign-off checklists.

**Notes**
- Timeline assumes parallel workstreams for UI, API, and data; adjust based on team size.
- MVP delivers core transactional flows; advanced analytics and automation can follow as Phase 2 enhancements.
