1 · Executive Summary

Fawnix Verse is a capable, feature-rich ERP platform with a sound high-level architecture — feature-per-service backend, per-service databases, a modern typed frontend. The risks are not architectural. They are in security hygiene, an incomplete product merge, the total absence of tests, and CI/CD that deploys unverified code straight to production.

Act first. Live Google/Gemini API keys are committed to git, and several services ship usable default credentials (JWT_SECRET, Admin@123, minioadmin). Rotate and remove before anything else.

The three themes

  • Incomplete HirePath → Fawnix merge. 8 backend services still under com.hirepath, missing exception handlers, residual artifacts; the "visitor management" frontend is a foreign body (own auth, own HTTP client, hardcoded ngrok URL, excluded from type-checking).
  • Maintainability drag. ~12% duplicated code — a security stack copy-pasted into 15 services, audit fields into 46 entities, DTOs across services — plus 1,000–4,000-line god-components.
  • No safety net. Zero tests, no CI gates, root containers, no resource limits, no rollback.

7 · Frontend & Backend Findings

Frontend (structural / standards)

  • God-components — task-management 4,129 lines / 28 useState / 4 file-level eslint-disable masking real effect-loop bugs. Decompose by view; use RHF+Zod for forms.
  • No data layer in migrated modules — inline useQuery keys collide; no hooks.ts/key-factory; no global staleTime. migrated
  • ~87 any, 25 no-op "use client", localStorage used as a business database (no versioning/TTL/conflict handling).

Backend (performance / structure)

  • HTTP calls inside @Transactional (inventory reserve, WhatsApp send) → split-brain state & pool exhaustion. Use @TransactionalEventListener(AFTER_COMMIT).
  • findAll().stream().filter() across every read path (TaskService visibility = 2 queries × N tasks). Push predicates to SQL, paginate, preload sets.
  • TaskService god-class (2,190 lines, 12 deps, 6 domains); file bytes stored as DB BLOB; money/timezone handling; broad/empty catches.
