# 2-Month MVP Plan (8 Weeks)

**Goal**
Deliver a usable MVP across Inventory, P2P, O2C, Task, Project, Asset, and Visitor modules in 8 weeks. Scope is intentionally thin-slice and focused on core transactions and reporting.

**Assumptions**
- 2-week sprints (4 sprints total).
- Team: 6-8 engineers, 1 PM/BA, 1 UX, 2 QA.
- Existing auth and org setup in `backend/services/identity-service/`.
- Service-per-domain architecture with REST APIs.

**MVP Scope by Module**
**Inventory**
- Item master (SKU, UoM, category, barcode).
- Warehouses and bins.
- Stock in/out/transfer, adjustments.
- Reorder minimum level alert.
- Basic valuation (weighted average).

**Procurement (P2P)**
- Vendor master.
- Purchase requisition -> PO -> goods receipt.
- Basic approval workflow (single-level).
- 2-way matching (PO vs invoice).

**Order-to-Cash (O2C)**
- Customer master and price list.
- Quote -> sales order -> shipment -> invoice.
- Payment capture (manual entry) and AR aging report.

**Task Management**
- Task CRUD, assignment, status, due dates.
- Basic Kanban and list views.
- Comments and attachments.

**Project Management**
- Project creation with phases and milestones.
- Task linkage to projects.
- Simple progress tracking (percent complete).

**Asset Management**
- Asset register with location and assignment.
- Transfer and disposal.
- Basic maintenance schedule with reminders.

**Visitor Management**
- Visitor pre-registration and approval.
- Check-in/out, host notification.
- Visitor log export.

**Out of Scope (Phase 2)**
- Advanced analytics, complex approval matrices, RFQ/RFP.
- 3-way matching, vendor scorecards, AR automation.
- Full maintenance work orders and spare parts.
- Multi-tenant billing and complex integrations.

**Timeline (8 Weeks)**
1. Week 1: Discovery, process mapping, finalize PRD, confirm MVP scope and KPIs.
2. Week 2: Platform foundations: RBAC updates, audit trail, notifications, document storage.
3. Week 3: Inventory core APIs and UI, stock transactions, item master.
4. Week 4: P2P flow (PR -> PO -> GRN), minimal approvals, vendor master.
5. Week 5: O2C flow (quote -> SO -> shipment -> invoice), AR aging report.
6. Week 6: Task + Project MVP, linkage of tasks to projects.
7. Week 7: Asset + Visitor MVP, basic dashboards.
8. Week 8: Hardening, UAT, performance checks, go-live readiness.

**Milestones**
- M1 (End Week 1): Approved PRD and MVP scope.
- M2 (End Week 2): Shared services ready.
- M3 (End Week 4): Inventory + P2P MVP.
- M4 (End Week 5): O2C MVP.
- M5 (End Week 7): Task/Project + Asset/Visitor MVP.
- M6 (End Week 8): UAT sign-off and launch.

**Critical Code Areas**
- `backend/services/inventory-service/` for inventory and P2P APIs.
- `backend/services/sales-service/` for O2C APIs.
- `backend/services/hrms-service/` for task and project modules.
- `backend/services/identity-service/` for RBAC and org hierarchy.
- `backend/services/platform/` for audit, workflow, notification primitives.
- `src/` for UI modules and dashboards.

**Risks and Mitigations**
- Scope creep: lock MVP features in Week 1, change control thereafter.
- Performance: early load tests in Week 6.
- Integration risk: establish API contracts in Week 2.

**Verification**
- Unit tests for domain logic per service.
- Integration tests for core flows:
- Inventory receipt -> stock update -> sales allocation.
- PR -> PO -> GRN -> invoice match.
- Quote -> SO -> shipment -> invoice -> payment.
- Task -> project milestone update.
- Asset transfer and maintenance reminder.
- Visitor pre-approval -> check-in/out.
- End-to-end UAT scripts with business sign-off.
